USING R STUDIO AND ITS COMMANDS

During this project we used R Studio, an integrated development environment (IDE) designed to work with the R programming language. This platform is widely used in statistics and environmental research because it allows users to write code, test commands and visualize the results of data analysis in the same workspace. In our case, RStudio was used to process and study air quality data in order to generate different graphs that help understand pollution behaviour over time.

Before starting the analysis it was necessary to prepare the working environment. For this reason, we first installed several libraries using the command install.packages(). Among them we used tidyverse, which contains tools for organizing and transforming datasets, dplyr which helps manipulate tables more easily, lares which includes functions to clean or replace text inside data, and finally openair, a package specifically developed for the analysis of atmospheric pollution and wind data. These libraries were then loaded into the session using the command library() so that their functions could be applied in the rest of the code.

Once the environment was ready, the next step was importing the dataset. This was done using read.csv(), which downloads the file and stores it inside a variable called city. However, the original dataset contained pollution values in a wide format, meaning that every hour of the day appeared in a different column. This structure is not ideal for visualization, so the command pivot_longer() was used to transform the data into a long format. After this transformation, the information was reorganized so that each row represented a single observation with three main variables: date, pollutant type and value. Some columns that were not necessary for the analysis were removed in order to keep the dataset simpler and easier to handle.

Another important step involved preparing the date and time information. Initially the date and hour were stored separately as text, so they were combined using the command mutate() together with paste0() to create a single column containing both elements. Because the time strings contained irregular formats, the function replaceall() from the lares package was used to standardize them. After cleaning the text, the column was converted into a real date-time format with as.POSIXct(), which allows R to interpret it correctly for time-series analysis. Additionally, the pollutant variable was converted into a factor using as.factor(), so that different pollutants could be grouped and represented more clearly in graphs.

With the data correctly prepared, we were able to generate visualizations using the functions provided by the openair package. One of the main commands used was timeVariation(), which creates a graphical summary showing how pollution levels vary depending on the hour of the day, the day of the week or the month of the year. Another function, trendLevel(), was applied to study the evolution of a specific pollutant such as H2S and to detect long-term patterns. These visualizations helped us interpret how air pollution changes over time and made it easier to communicate the results of our research.
