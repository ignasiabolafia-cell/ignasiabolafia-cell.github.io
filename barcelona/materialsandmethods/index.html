USING R STUDIO AND ITS COMMANDS

In this project we have worked with R Studio, an integrated development environment (IDE) created specifically to use the R programming language. It is widely used for statistical analysis and data science. One of its main advantages is that it allows users to write code, debug programs, and visualize results in the same working environment.

To obtain the results for our research, we applied different commands. This page explains the purpose of each command that we used until we were able to generate the final graphics and visualizations.

1. ENVIRONMENT SETUP

These commands prepare RStudio by installing and loading the tools required to analyse air quality datasets.

  • install.packages(c("openair","tidyverse","lares")): Downloads the necessary libraries from the internet so they can be used inside the project.
  • library(tidyverse): Loads a collection of tools that help with data cleaning, transformation and organization.
  • library(dplyr): Loads a component of the tidyverse focused on manipulating and arranging data tables.
  • library(lares): A package used to replace text values and clean irregular strings inside datasets.
  • library(openair): Specialized package designed to perform air pollution and meteorological data analysis.

2. DATA IMPORT AND RESHAPING

Air pollution databases usually come in a wide format, meaning that every hour of the day is stored in a different column. In order to analyze the information properly, we convert it to a long format, where every observation appears in a separate row.

  • city <- read.csv("..."): Downloads the CSV dataset from the internet and stores it in a variable called city.
  • pivot_longer(city, cols=c(h01...h24), names_to="hour", values_to="value"): Transforms the twenty-four hourly columns into two new columns called hour and value.
  • city2 <- city1[-c(1,2,4,6:16)]: Removes unnecessary columns so the dataset becomes simpler and easier to manage.

3. DATE AND TIME FORMATTING

Computers initially interpret dates as plain text. For proper time-series analysis, we must convert them into real date-time objects.

  • mutate(date = paste0(data, " ", hour)): Creates a new column that merges the date and hour into a single variable.
  • replaceall(city2, ...): Cleans irregular time strings and standardizes the time format.
  • as.POSIXct(city2$date, ..., tz="Europe/Madrid"): Converts the date column into a proper datetime format that R can interpret.
  • as.factor(city2$pollutant): Transforms pollutant names into categorical variables so they can be grouped in charts.

4. ADVANCED VISUALIZATION (openair)

The following functions from the openair package automatically generate environmental graphs that help interpret air quality patterns.

  • timeVariation(city2): Displays how pollution levels change depending on the hour of the day, day of the week and month.
  • trendLevel(city2, pollutant="H2S"): Creates a visual heatmap that helps identify long-term pollution trends.
